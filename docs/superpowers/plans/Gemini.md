## 2. Orchestrator Strategy: Moving Beyond Linear Steps
Your current 7-step plan is **linear**. 2026 best practices suggest an **Agentic Loop** (Observe → Plan → Act → Learn).

### Missed: The "Critic" Layer
For EU funds, accuracy is non-negotiable.
* **Recommendation:** Implement a **Planner-Critic-Executor** pattern.
* **How:** Between Step 6 (Plan) and Step 7 (Build), add an automated **Compliance Agent** that specifically checks the draft against the call's eligibility criteria and flags "Auto-Rejection" risks before the user even sees the first draft.

### Missed: State Persistence (Memory)
* **The Issue:** Long orchestrator runs often fail mid-way due to session drops.
* **2026 Standard:** Use **Redis for Short-term memory** (current step variables) and **PostgreSQL JSONB for Long-term "Project Memory."** Ensure your SSE re-connection logic can "replay" the AI's thought process if the user refreshes during Step 7.

## 3. Streaming & UX Audit
You are currently using **SSE (Server-Sent Events)**.

* **Audit:** SSE is great for 2026 "Read-Only" streaming (like the AI writing a section).
* **Missing:** **Bi-directional interaction.** If the AI is writing a 20-page project and the user notices a mistake on page 2, they shouldn't wait for page 20 to finish.
* **Upgrade Path:** Consider **OpenAI Realtime API (WebSocket)** for the "Build" phase. It allows the user to interrupt the AI mid-stream to correct a budget figure, significantly reducing "Hallucination Waste."
